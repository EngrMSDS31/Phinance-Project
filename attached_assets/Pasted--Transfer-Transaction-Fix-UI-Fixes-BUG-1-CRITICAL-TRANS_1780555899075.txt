**BUG 1 — CRITICAL: TRANSFER TRANSACTIONS DO NOT MOVE MONEY BETWEEN CASH AND HOLDINGS**

The Transfer: Cash to Holding and Transfer: Holding to Cash transactions save correctly but the money does not actually move between Total Cash and Current Value.

**What must happen:**

**Transfer: Cash to Holding**
- Total Cash decreases by the transferred amount
- Current Value (holdings market value) increases by the transferred amount
- Total Portfolio Value stays the same (money moved internally, not added or removed)
- The change must reflect immediately in the Portfolio detail screen AND on the Dashboard (Total Cash card, Current Value card, and Portfolio Allocation pie chart)

**Transfer: Holding to Cash**
- Current Value decreases by the transferred amount
- Total Cash increases by the transferred amount
- Total Portfolio Value stays the same
- Same immediate reflection in Portfolio and Dashboard

**How to fix:**

1. Find the function that processes Transfer transactions when saved
2. Check if it is actually updating both the cash balance AND the holding value — it is likely only recording the transaction without updating either balance
3. For Cash to Holding transfers: deduct the amount from the portfolio's cash balance AND add it to the target holding's value
4. For Holding to Cash transfers: deduct the amount from the holding's value AND add it to the portfolio's cash balance
5. Make sure the Dashboard recalculates Total Cash and Current Value after a transfer saves
6. Make sure the Portfolio Allocation pie chart updates to reflect the new values
7. Test Cash to Holding: start with ₱770,000 cash → transfer ₱100,000 to MP2 holding → Total Cash becomes ₱670,000 → Current Value becomes ₱100,000 → Total Portfolio Value stays ₱770,000
8. Test Holding to Cash: reverse the above → Current Value goes back to ₱0 → Total Cash goes back to ₱770,000
9. Confirm Dashboard reflects both tests correctly including the pie chart
