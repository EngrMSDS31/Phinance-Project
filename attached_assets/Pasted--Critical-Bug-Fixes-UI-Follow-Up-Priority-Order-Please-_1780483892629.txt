**PRIORITY 1 — CRITICAL BUG: Edit & Delete Broken for All Transaction Types**

This is the most important fix. Currently, editing and deleting transactions fails in multiple places across the app. This must work universally without exception.

**The rule is absolute:** Every single transaction record in the app — regardless of type, source, portfolio, or market — must be fully editable and deletable by the user.

**Scope — applies to ALL of the following without exception:**
- All transaction types: Buy, Sell, Dividend, Deposit, Withdrawal, Other Income, Other Expenses
- All portfolio types: Regular stock portfolios, Custom portfolios, Fund portfolios, Bond portfolios, Mixed portfolios
- All markets: PSE, US, LSE, and any Custom/Fund/Bond/Mixed markets
- All import methods: Manually entered transactions AND CSV-imported transactions
- All locations where transactions appear: Portfolio Transactions list, Dividend Calendar View, Dividend Calendar List View, and any other screen that shows transaction records

**Root cause investigation — check all of the following:**

1. **Missing transaction ID:** Some transactions (especially imported ones or custom asset transactions) may not have a properly stored `id` field. Without an ID, the edit/delete API call has nothing to reference and silently fails or returns an error. Fix: ensure every transaction saved to the database has a unique ID, and that the ID is passed correctly in all edit/delete requests.

2. **Wrong API endpoint or method:** The edit/delete calls may be hitting the wrong route for certain transaction types (e.g. dividend transactions may use a different endpoint than buy/sell). Fix: verify that the correct endpoint is called for each transaction type, and that HTTP method (PUT/PATCH for edit, DELETE for delete) is correct.

3. **Schema mismatch on save:** The edit form may be sending fields that don't match the database schema for certain transaction types (as seen with the MP2 "Failed to update" error). Fix: ensure the payload sent on save maps correctly to the stored schema for every transaction type, including custom/fund/bond transactions which may have different field names.

4. **Read-only flag on imported transactions:** Some apps mark imported transactions as read-only. If this exists, remove it entirely. No transaction should ever be locked from editing or deletion.

5. **Missing error handling:** Wrap every edit and delete operation in proper try/catch. If an operation fails, show a specific error message explaining what went wrong (not just "Failed to update"). Log the full error to the console for debugging.

**After fixing, verify these specific cases work:**
- Edit a manually added Buy transaction in a stock portfolio → save successfully
- Edit a CSV-imported Dividend transaction → save successfully
- Edit an MP2 custom portfolio Dividend transaction → save successfully (the "Failed to update" bug)
- Delete any transaction in the Dividend Calendar List View → deleted and UI refreshes
- Delete any transaction in the Dividend Calendar View (Calendar) → deleted and calendar refreshes
- Edit/Delete a transaction in a Fund, Bond, or Custom market portfolio → works correctly
- After any edit or delete, all affected portfolio metrics (Total Profit, Current Value, Dividends, etc.) recompute and reflect the change immediately
