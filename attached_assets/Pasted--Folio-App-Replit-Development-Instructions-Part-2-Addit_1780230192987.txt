## 1. ANALYTICS SECTION — RECALCULATIONS & FIXES

### 1.1 Metric Card: "CURRENT VALUE"

**New formula:**
```
Current Value = Total Assets + Total Profit
```

Where:
- **Total Assets** = Sum of (current market price × quantity) for all holdings with quantity > 0, converted to base currency.
- **Total Profit** = As defined in section 1.2 below (Capital Gain + Realized Gains + Dividends — no Cash Incentive).

Sub-label below the value:
```
Invested: ₱X,XXX.XX  (+XX.X%)
```
Where:
- **Invested** = Total Cost = Sum of (average cost per share × current quantity) for all holdings, converted to base currency. This is the actual amount of capital deployed into assets.
- **P&L %** = (Current Value − Invested) ÷ Invested × 100

> **Note to Replit:** "Invested" here is the total cost basis of currently held positions only — not historical total spent including sold shares. Use: `SUM(holding.avgCost * holding.quantity)` across all holdings.

---

### 1.2 Metric Card: "TOTAL PROFIT"

**Remove "Cash Incentive"** from Total Profit entirely. It must no longer appear in this card anywhere — not as a line item, not in the total calculation.

**New formula:**
```
Total Profit = Capital Gain + Realized Gains + Dividends
```

Where:
- **Capital Gain** = Sum of (current price − avg cost) × quantity for all open positions (can be negative if unrealized loss).
- **Realized Gains** = Sum of profit/loss from all fully or partially closed (sold) positions.
- **Dividends** = Sum of all Net Dividend amounts (Gross Dividend − Withholding Tax) recorded across all dividend/distribution/interest transactions.

Display breakdown inside the card exactly as:
```
Capital Gain:    ₱X,XXX.XX   (red if negative, green if positive)
Realized Gains:  ₱X,XXX.XX   (green)
Dividends:       ₱X,XXX.XX   (green)
```

Do NOT add a Cash Incentive line here. That value moves to the Portfolio section's Total Cash card (see Section 3.1 below).

### 1.3 Metric Card: "DIVIDEND YIELD" — Verify & Fix All Sub-Values

The following four values in the Dividend Yield card must be calculated correctly. Replit should implement each formula as described:

#### Dividend Yield (headline %)
```
Dividend Yield = (Total Annual Dividends ÷ Current Value) × 100
```
- Total Annual Dividends = Sum of all net dividends received in the last 12 months (rolling, not calendar year).
- Current Value = same as card 1 above (Total Assets + Total Profit).

#### Yield on Cost
```
Yield on Cost = (Total Annual Dividends ÷ Total Invested Cost) × 100
```
- Total Invested Cost = Total Cost (avg cost × quantity summed across all holdings).
- Total Annual Dividends = same as above (last 12 months net dividends).

#### Annual Passive Income (previously "Annual Div")
```
Annual Passive Income = Total Annual Dividends (last 12 months net dividends, in base currency)
```
- This is the raw PHP/currency amount, not a percentage.
- Label it "Annual Passive Income" (rename from "Annual Div").

#### Growth (5Y)
```
5-Year Dividend Growth = ((Dividends in most recent year ÷ Dividends 5 years ago) ^ (1/5) − 1) × 100
```
- "Most recent year" = last full calendar year with dividend data.
- "5 years ago" = the calendar year 5 years prior.
- If fewer than 2 years of dividend data exist: display "N/A" instead of a percentage.
- If dividends 5 years ago were ₱0: display "N/A" (cannot divide by zero).

> **Verification checklist for Replit:**
> - Pull the raw dividend transaction records and manually sum the last 12 months. Confirm it matches "Annual Passive Income."
> - Divide that sum by Current Value — confirm it matches "Dividend Yield %."
> - Divide that sum by Invested Cost — confirm it matches "Yield on Cost %."
> - For Growth 5Y: pull yearly totals for the past 5+ years and apply the CAGR formula.

### 1.4 Dividends Bar Chart — Fix Values for All Three Toggles

**Current issue:** Bar values are incorrect — they do not match the actual dividend records.

**Required fix — correct formula for each toggle:**

#### Monthly View
```
Bar height for Month M = SUM of Net Dividends received in Month M of the selected year

Net Dividend = Gross Dividend Amount − Withholding Tax Amount
```
- Source: All transactions of type Dividend / Distribution / Coupon / Interest / Staking Reward within that month.
- Year is controlled by the ← YYYY → year navigator.
- All 12 month bars must always render (months with no dividends = bar height 0, not hidden).
- Convert all amounts to base currency before summing (apply exchange rate at time of transaction).

#### Yearly View
```
Bar height for Year Y = SUM of Net Dividends received across all months of Year Y
```
- Show at minimum: all years from the earliest dividend transaction to the current year.
- No year filter needed — show all years with a horizontal scroll if needed, or show last 5–7 years.

#### Asset View
```
Bar height for Asset A = SUM of all Net Dividends ever received from Asset A (all time)
```
- Group by holding/symbol, not by portfolio.
- Sort bars descending by total net dividend amount (largest first).
- If a year filter is active (from the year navigator), apply it: show only dividends from that year per asset.

**Implementation note for Replit:**
```javascript
// Pseudocode for net dividend calculation
function getNetDividend(transaction) {
  const gross = transaction.totalAmount || (transaction.pricePerShare * transaction.shares);
  const tax = transaction.withholdingTax || 0;
  return gross - tax;
}

// Monthly aggregation
function getMonthlyDividends(year) {
  const months = Array(12).fill(0);
  transactions
    .filter(t => isDividendType(t) && t.date.getFullYear() === year)
    .forEach(t => {
      const month = t.date.getMonth(); // 0-indexed
      months[month] += convertToBaseCurrency(getNetDividend(t), t.currency, t.date);
    });
  return months;
}
```

**Verification step:** The sum of all bars in Monthly view for a given year must equal the same year's bar in Yearly view. If they don't match, there is a bug in the aggregation or currency conversion.

### 3.3 Custom Asset Transaction Logic — "Deposit as Asset" Flow

**Current issue:** For custom assets like Government Bonds, the only transaction options are investment-style ones (Buy, Sell, Dividend, etc.). When the user deposits cash into a bond account, there is no way to classify that deposit as an asset that contributes to Current Value. The deposit currently behaves like idle cash, not like an asset.

**Required behavior — "Deposit as Asset" classification:**

For **custom asset types** (specifically: Savings Account, Bonds, and Other Custom Assets), a **Deposit** transaction should be treated as **adding to the asset value** — not as adding to idle cash.

Here is the exact logic Replit must implement:

#### How Deposits on Custom Assets Work:

1. **When a user adds a "Deposit" transaction to a custom asset holding:**
   - The deposited amount increases the **Current Value** of that holding.
   - It is classified as an **asset contribution**, not idle cash.
   - Formula: `holding.currentValue += depositAmount`
   - The holding shows up in the **Current Value** card (assets-only total).

2. **Idle Cash vs. Asset Cash distinction:**
   - Idle Cash = cash balance inside a portfolio that is NOT assigned to any holding (free-floating uninvested cash).
   - Asset Cash = cash deposited into a specific custom holding (Savings, Bond, etc.) — this is considered invested/deployed, even though it earns interest rather than capital gains.
   - When the user deposits into a custom holding, the idle cash balance must decrease by that amount (the money moves from idle to the asset).

3. **How to reflect returns (interest/dividends) for custom assets:**
   - When a "Dividend/Interest" or "Other Income" transaction is added to a custom asset holding, it contributes to **Total Profit → Dividends** (or a separate "Income" line if preferred).
   - These income transactions do **not** increase the asset's principal/current value — they are profit only.

4. **What appears in each Portfolio card for custom assets:**

| Card | What it shows for custom asset holdings |
|---|---|
| Total Portfolio Value | Includes custom asset deposits (principal) + any income |
| Current Value | Includes custom asset current value (principal deposited) |
| Total Profit | Includes income/interest earned on the custom asset |
| Total Cash | Only shows true idle cash — does NOT include custom asset deposits |

5. **Transaction type changes for Bonds specifically:**

For holdings with asset type **Bonds**, the transaction dropdown must include:
```
TRADE group:
  Buy          ← For purchasing bond units on secondary market
  Sell         ← For selling bond units

INCOME group:
  Coupon / Interest    ← Regular coupon payments
  Maturity             ← Principal returned at maturity
  Other Income

CASH group:
  Deposit      ← For depositing principal into bond account (treated as asset value)
  Withdrawal   ← For withdrawing from bond account

EXPENSE group:
  Other Expenses
```

The key addition is **Deposit** in the CASH group for Bonds — so the user can deposit into their bond account and have it count as an asset.

6. **Example flow for Government Bonds (for Replit to follow):**
```
User has: Idle Cash = ₱100,000 in "Pagibig MP2" portfolio
User adds Deposit transaction on "Government Bond" holding = ₱50,000

Result:
  Idle Cash:           ₱50,000 (reduced from ₱100,000)
  Government Bond (Current Value): ₱50,000 (increased from ₱0)
  Total Portfolio Value: ₱100,000 (unchanged — money moved, not added)
  Current Value card:  ₱50,000 (bond is now an asset)
  Total Cash card:     ₱50,000 (remaining idle cash)

Later: Coupon Interest = ₱1,500 added as "Coupon/Interest" transaction on the bond

Result:
  Total Profit → Dividends: +₱1,500
  Government Bond (Current Value): ₱50,000 (principal unchanged)
  Total Portfolio Value: ₱101,500
```

### 3.4 "Hide Values" Global Icon — Fix Scope

**Current issue:** The eye/hide icon in the top navigation bar (visible in screenshots as a strikethrough-eye icon) currently only hides the transaction list in the Portfolio section.

**Required fix:** The hide icon must hide **ALL numerical values** across the entire app — every number on every screen.

**Scope of values to hide/show:**
- Dashboard: All 4 metric card values, pie chart legend values and percentages.
- Analytics: All metric card values, chart axis values, chart data labels, percentage breakdowns.
- Dividend Calendar: All amount values in List view (Amount column), all values in the detail popup.
- Portfolio: All metric card values (Total Portfolio Value, Total Profit, Current Value, Total Cash), all individual holding values and percentages, all transaction amounts.
- Any other numerical value visible anywhere in the app.

**How to hide:** Replace each numerical value with `••••••` (6 bullet characters) or `****` — do not leave blank spaces.

**Implementation approach:**
```javascript
// Global state
const [valuesHidden, setValuesHidden] = useState(false);

// Toggle function — fired by the eye icon in top nav
function toggleHideValues() {
  setValuesHidden(prev => !prev);
}

// Helper used everywhere a value is rendered
function displayValue(value, format = 'currency') {
  if (valuesHidden) return '••••••';
  return formatValue(value, format); // your existing formatter
}
```

- Store the `valuesHidden` state in a global context (React Context or state management store) so all components can access it.
- The eye icon must update its visual state: open eye = values visible, crossed-out eye = values hidden.
- State must persist across tab/section navigation within the same session (but can reset on full app reload — persistence between sessions is optional).
