**Fix:**

2. The Calendar View and List View must read dividend data directly from the transaction records in the database, not from a separate dividend schedule or ex-dividend table
3. Implement a single-source-of-truth: one dividend transaction record in the database, displayed in all three views
4. **Sync edits and deletes across all three views:**

   - If I edit a dividend in List View → Transaction History and Calendar View update immediately

**BUG 3 — UI FIX: TOTAL CASH DISPLAY IN PORTFOLIO SECTION AND DASHBOARD**

**Problem:** The Total Cash section currently shows the cash balance value. The display needs to change to show the difference between Total Portfolio Value and Current Value (market value of holdings).

**Formula for display:**
```
Total Cash displayed = Total Portfolio Value − Current Value (holdings market value)
```

**Example:** Total Portfolio Value = ₱100,000 and Current Value (holdings) = ₱90,000 → Total Cash displays ₱10,000

**Fix:**
1. This is a **display change only** — do not change any calculations or stored data
2. In the Portfolio detail screen (where Total Profit, Current Value, and Total Cash cards are shown), change the Total Cash card to display: Total Portfolio Value minus Current Value
3. Apply the same display change to the Total Cash section on the Dashboard
4. The subtext formatting and color coding remain the same
5. Test: Verify the displayed Total Cash equals Total Portfolio Value minus the sum of all holdings' current market values
